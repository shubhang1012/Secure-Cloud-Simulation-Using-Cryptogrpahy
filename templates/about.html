Secure File StorageSystem

File Storage Using Hybrid Cryptography



Objective: Using hybrid cryptography, a platform that users can rely on is created for cloud file storage.

Methodology:

  1. Load the file on the server.
  2. Dividing the uploaded file into N parts depending on the fixed block size.
  3. Encrypting all the sub-files using any one of the selected algorithms
    (Algorithm is changed with every part in round robin fashion).
  4. The public key for the cryptography algorithm is given to the user and must be securely transferred to the recipient. The keys for the employed cryptography algorithms are kept in a file. The resulting file is then secured using a different cryptographic algorithm.
    (Currently, we are using symmetric encryption.)

To restore:


  1. Load the key (.pem file) on the server. Ask it from your sender.
  2. Decrypt the keys of the algorithms.
  3. Using the same algorithms that were used to encrypt them, decrypt each of the N subfiles.
  4. The user will be able to download the original file, which is created by combining all N subfiles.
